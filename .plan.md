### Phase 1: RED/YELLOW/GREEN Flow (8-10 hours)

**Goal:** Core workflow enforcement

#### Tasks:

1. **RED LIGHT command** (2.5 hours)
   ```go
   yo red           // Create file, open $EDITOR
   yo red -i        // Interactive mode
   yo verify red    // Validate completion
   ```
   
   Interactive prompts:
   - Problem description (text input)
   - Impact (multi-select checkboxes)
   - Severity (P0/P1/P2/P3 select)
   
   Validator:
   - Check markdown has problem statement
   - At least 1 impact checked
   - Severity selected
   
   Output: `.yo/current_task.md` with RED LIGHT section filled

2. **YELLOW LIGHT command** (3 hours)
   ```go
   yo yellow        // Create/append to file, open $EDITOR
   yo yellow -i     // Interactive mode
   yo verify yellow // Validate completion
   ```
   
   Interactive prompts:
   - Root cause (3 levels: immediate/underlying/system)
   - Solution options (enter 3 minimum)
   - Time estimates for each
   - Decision (which option A/B/C)
   - Implementation steps
   - Success criteria (list)
   
   Validator:
   - 3+ solution options present
   - Time estimate exists
   - Success criteria (at least 2)
   - Decision made
   
   Output: Current task now has YELLOW LIGHT section complete

3. **GREEN LIGHT command** (2.5 hours)
   ```go
   yo go            // Start timer, change stage
   yo go --time 5h  // Override time estimate
   ```
   
   Actions:
   - Verify RED + YELLOW complete
   - Extract time estimate from YELLOW LIGHT
   - Update state.json: stage="green", timer starts
   - Start timer goroutine (in-memory)
   - Show notification: "ğŸŸ¢ GREEN LIGHT - Timer started: Xh Ym"
   - Log to activity: stage_change event

4. **Verification logic** (2 hours)
   - Markdown parser
   - Checkbox detection (regex for [x] or [X])
   - Section extraction
   - Validation error messages

**Deliverable:** Can complete RED â†’ YELLOW â†’ GREEN flow

### Phase 2: Timer System (4-6 hours)

**Goal:** Count-up timer with threshold notifications

#### Tasks:

1. **Timer goroutine** (2 hours)
   - Start counting from 0
   - Track elapsed time
   - Calculate progress % vs threshold
   - Persist timer state every 1 minute
   
2. **Timer notifications** (1.5 hours)
   - At 100% of estimate: "â±ï¸ Hit your Xh estimate. Still on track?"
   - At 150%: "âš ï¸ 50% over estimate"
   - At 200%: "ğŸš¨ Doubled your estimate"
   - Don't repeat notifications (track in state)

3. **Extension command** (1 hour)
   ```go
   yo extend 1h "API more complex than expected"
   ```
   - Add to state.extensions[]
   - Increase threshold
   - Show updated threshold
   - Log to tech_debt_log.md automatically

4. **Timer display** (1.5 hours)
   ```go
   yo status
   yo timer  // Timer-only view
   ```
   - Show: 2h 30m / 4h (63%)
   - Show: 5h 15m / 4h (131%) if over
   - Color coding (green/yellow/red)

**Deliverable:** Timer counts up, notifies at milestones

---

### Phase 3: File Watcher (6-8 hours)

**Goal:** Track all file activity in watched directories

#### Tasks:

1. **Git repo detection** (1.5 hours)
   - Scan watch_dirs (from config)
   - Find all directories with .git
   - Store repo paths
   - Refresh on `yo watch` start

2. **File watcher daemon** (3 hours)
   ```go
   yo watch         // Start watcher (blocks terminal)
   yo watch --bg    // Background daemon
   yo watch stop    // Stop background daemon
   ```
   
   Implementation:
   - Use fsnotify to watch all repos
   - Debounce rapid saves (1 second window)
   - Detect which repo file belongs to
   - Check if repo matches current task
   - Log to activity.jsonl
   
   Events logged:
   - file_change: every save
   - Mark as untracked if repo != current_task_repo

3. **Activity query** (2.5 hours)
   ```go
   yo activity              // Today
   yo activity --yesterday
   yo activity --week
   yo activity --repo myapp
   ```
   
   Display:
   ```
   Today (4h 30m total):
     myapp       3h 45m (current task) âœ…
     side-proj   30m (untracked)
     dotfiles    15m (untracked)
   
   Files touched (use --verbose):
     myapp: src/deploy.js, src/api.js, tests/
   ```
   
4. **Focus score calculation** (1 hour)
   ```go
   yo focus
   ```
   - Parse activity.jsonl for today
   - Sum time per repo
   - Calculate: time_on_task / total_time * 100
   - Show breakdown

**Deliverable:** File watcher logs all activity, can query it

### Phase 4: Task Completion (5-7 hours)

**Goal:** Two types of done - task and session

#### Tasks:

1. **Task completion** (3 hours)
   ```go
   yo done
   ```
   
   Flow:
   - Verify current stage is "green"
   - Load success criteria from current_task.md
   - Interactive checklist:
     ```
     Success criteria:
       [?] User can click button, workflow deploys (y/n) > y
       [?] Status updates correctly (y/n) > y
       [?] Error messages readable (y/n) > y
     ```
   - If all yes:
     - Stop timer
     - Calculate actual vs estimated time
     - Show accuracy: "Actual: 4.5h | Estimated: 4h | Accuracy: 112%"
     - Archive to done/YYYY-MM-DD_task_name.md
     - Log completion to activity.jsonl
     - Clear current_task.md
     - Update state.json: stage="none"
     - Ask: "Pick next task? (y/n)"
   - If any no:
     - Ask: "Mark incomplete and continue? (y/n)"

2. **Session end** (2 hours)
   ```go
   yo off
   ```
   
   Flow:
   - Calculate session duration
   - Parse activity.jsonl for this session
   - Group by repo
   - Calculate focus score
   - Show summary:
     ```
     Session summary:
       Duration: 4h 30m
       Primary: myapp (3h 45m) - current task
       Untracked: side-proj (30m), dotfiles (15m)
       Focus: 83%
     
     Current task incomplete. Roll to tomorrow? (y/n) > y
     ```
   - Log session_end to activity.jsonl
   - Save session summary to sessions/
   - Update state: session.active = false
   - Ask: "Any new issues for backlog? (y/n)"

3. **Next task picker** (2 hours)
   ```go
   yo next
   ```
   
   Flow:
   - Parse backlog.md
   - Show P0 tasks first, then P1
   - Interactive select:
     ```
     Pick next task:
       [1] Deploy button missing (P0)
       [2] Auth flow broken (P0)
       [3] Status polling (P1)
       [q] Quit
     > 1
     ```
   - Move selected task to current_task.md as new RED LIGHT
   - Ask: "Start RED LIGHT now? (y/n)"

**Deliverable:** Can complete tasks and sessions properly
